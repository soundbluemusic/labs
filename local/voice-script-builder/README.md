# 한국어 보이스 클로닝 대본 생성기 — 로컬 클론

`labs` 사이트에 임베드된 claude.site 아티팩트
(`voice-script-builder`, artifact `a956aeb4-…`)를 **로컬에서 단독 실행**할 수 있게
옮겨 둔 복제본입니다. 빌드 단계 없이 브라우저에서 바로 열립니다.

## 파일

| 파일 | 내용 |
| --- | --- |
| `source.jsx` | 원본 아티팩트 소스(참고/보존용, 그대로 추출) |
| `index.html` | 로컬 실행용 단독 클론 (React/Babel/Tailwind를 CDN으로 로드) |

## 실행 방법

API 호출(CORS) 때문에 `file://`로 바로 열기보다 간단한 정적 서버로 띄우는 걸 권장합니다.

```bash
cd local/voice-script-builder
python3 -m http.server 8000
# 브라우저에서 http://localhost:8000 접속
```

처음 "스크립트 생성"을 누르면 **Anthropic API 키(`sk-ant-…`)** 를 물어봅니다.
키는 이 브라우저의 `localStorage`에만 저장되며, Anthropic API 외 다른 곳으로 전송되지 않습니다.
(키를 바꾸려면 콘솔에서 `localStorage.removeItem('anthropic_api_key')` 후 새로고침.)

## 왜 로컬에서는 API 키가 필요한가

원본은 claude.ai 아티팩트 환경 안에서 돌아가며, 그 환경의 **인증 프록시**가
`api.anthropic.com` 호출에 자동으로 인증을 붙여 줍니다. 그래서 사이트에서는
"로그인"만 하면 동작합니다. 로컬에는 그 프록시가 없으므로, 클론에서는
사용자 키를 헤더(`x-api-key` + `anthropic-dangerous-direct-browser-access`)에
직접 주입하도록 **그 부분만** 수정했습니다. 나머지 로직은 원본과 동일합니다.

## 구조: Claude가 필요한 부분 vs 순수 알고리즘

이 툴은 두 단계로 나뉘며, **무거운 선택 로직은 이미 Claude 없이 동작**합니다.

| 단계 | 함수 | Claude 필요? |
| --- | --- | --- |
| 자연스러운 A/B 대화 **문장 생성** | `callAPI` / `buildPoolPrompt` | ✅ 필요 (창작) |
| 대화 텍스트 파싱·검증(반응성/음절수) | `parseDialogues` / `hasReaction` | ❌ 순수 알고리즘 |
| 한글 초/중/종성 분해 | `decompose` | ❌ 순수 알고리즘 |
| 음소 빈도 벡터화 | `phonemeVector` | ❌ 순수 알고리즘 |
| 음소 커버리지 최대화 200개 선택 | `greedySelect` | ❌ 순수 알고리즘 |

즉 Claude는 오직 **"새 대화 문장을 짓는" 단계**에만 쓰입니다.
선택·커버리지 계산은 전부 코드입니다.

## "로그인 없는 알고리즘 버전"으로 가는 길

원본에는 이미 `loadPool(JSON)` — 저장해 둔 대화 풀을 불러와 선택만 수행 — 기능이 있습니다.
따라서 **로그인 없는 버전**은 두 갈래로 만들 수 있습니다.

1. **큐레이션 코퍼스 + 기존 그리디 선택 (권장)**
   미리 만들어 둔 자연스러운 대화 풀(JSON)을 동봉하고, `greedySelect`로 음소 커버리지를
   채워 매번 다른 200개를 뽑습니다. 문장 품질은 코퍼스 품질에 달려 있고, 실행 시 LLM 호출 0회.
2. **템플릿 기반 합성 생성**
   문장 템플릿 + 어휘 슬롯으로 대화를 조립. 완전 자율이지만 희귀 음소
   (ㅃ·ㄸ·ㅒ·ㅖ·ㅢ, 겹받침 ㄳ·ㄵ·ㄺ 등)를 자연스럽게 채우기가 어렵습니다.

→ 다음 단계로 어느 방향을 원하는지 알려주시면 그쪽으로 진행하겠습니다.
